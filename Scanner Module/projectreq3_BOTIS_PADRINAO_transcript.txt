BOTIS CODE:
Scanner Module\Botis\projectreqt3_BOTIS_PADRINAO.java

BOTIS' APPROACH: 

Botis used a manual pointer (cursor) to scan the code character by character, 

skipping over whitespace and comments while keeping track of line numbers. 

Botis' logic uses a Switch-Case system to immediately identify symbols, and "lookahead" functions to check if characters 

like = are actually part of a larger operator like ==. Finally, it groups characters into lexemes, automatically labeling 

them as keywords, numbers, or strings based on C# rules.

PADRINAO CODE:
Scanner Module\Padrinao\projectreq3_BOTIS_PADRINAO.java

PADRINAO's APPROACH:

Padrinao created a scanner that reasds the source code and separes the tex into different tokens.

Each token is stored with its type, value, and the line number where it was found.

The program first removes all spaces and comments, including single-line (//) and multi-line (/* ... */) comments.

After that, the scanner checks the remaining text and identifies keywords, identifiers, numbers, strings, characters, operators, and delimiters.

The program checks longer operators first before checking single symbols to make sure they are recognized correctly.

If the scanner finds a symbol that does not belong to the language rules, it marks it as an illegal token and shows the line where it occurred.
